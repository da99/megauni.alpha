route_it( opts = {} )
============================================================

Method to help prevent typos from having to repeat the same regular expression over and over again.

If :from is a string, then any beginning or ending forward slashes are removed and replaced by "^/" and "/?$".

It can get confusing using ’%d’ in :to (as described in Ramaze routing docs). That‘s why you can use any of the following values to prevent typos: 

  %d   \d   d

All of the above values are turned into ’%d’ (without quote marks) when used in :to as a String.

Example: 

  route_it( :from => '/logout',         :to => :delete )
  route_it( :from => 'newspaper/(\d+)', :to => R(self, :index, 'd') )
  route_it( :from => 'newspaper/toggle_privacy/', :to => :toggle_privacy )
  route_it( :from => :update_privacy, :to => :toggle_privacy )
